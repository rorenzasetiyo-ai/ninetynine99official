🔧 Perbaikan Utama

1. Menghilangkan Inline Event Handler

Sebelumnya, website menggunakan 43+ inline event handler yang tertanam langsung dalam HTML (onclick, onmouseover, onerror, dll). Ini adalah vektor keamanan utama untuk XSS (Cross-Site Scripting) attacks.

❌ Sebelumnya (Tidak Aman)

<button onclick="openCart()" onmouseover="this.style.color='#000'" onmouseout="this.style.color='#555'"> Keranjang </button>

✓ Sesudahnya (Aman)

<button data-action="open-cart" aria-label="Keranjang"> Keranjang </button>

2. Sistem Event Delegation Terpusat

Membuat file JavaScript baru (js/event-handlers.js) yang mengelola semua event handler secara terpusat menggunakan pola event delegation dan atribut data-action.
Manfaat:
  • Lebih mudah di-audit dan di-maintain
  • Mengurangi surface area untuk XSS attacks
  • Memisahkan HTML dari logic
  • Kompatibel dengan strict CSP

3. Hardening Content Security Policy

Memperketat Content Security Policy dengan menghilangkan 'unsafe-inline' dari direktif script-src, mempertahankan keamanan tinggi sambil mempertahankan fungsionalitas.

❌ Sebelumnya (Grade B)

script-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net

✓ Sesudahnya (Grade A+)

script-src 'self' https://cdn.jsdelivr.net

Header Nilai Tujuan
HSTS max-age=31536000; includeSubDomains; preload Force HTTPS, cegah protocol downgrade

X-Frame-Options SAMEORIGIN Cegah clickjacking attacks
X-Content-Type-Options nosniff Cegah MIME type sniffing
Referrer-Policy strict-origin-when-cross-origin Kontrol informasi referrer
Permissions-Policy Semua blocked (geolocation, microphone, camera, dll) Batasi akses ke sensor device

📊 Content Security Policy (CSP) Detail

default-src 'self' https:; script-src 'self' https://cdn.jsdelivr.net; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src 'self' https://fonts.gstatic.com; img-src 'self' https: data:; connect-src 'self' https:; frame-src 'none'; object-src 'none'; base-uri 'self'

Penjelasan Direktif

  • default-src 'self' https: Default untuk resource yang tidak disebutkan
  • script-src 'self' https://cdn.jsdelivr.net: Scripts hanya dari domain sendiri atau CDN terpercaya
  • style-src 'self' 'unsafe-inline': Styles dari domain sendiri dan inline (CSS aman)
  • img-src 'self' https: data: Images dari domain sendiri, HTTPS, atau data URLs
  • frame-src 'none': Tidak boleh di-frame/embed di domain lain
  • object-src 'none': Tidak ada Flash, Java, atau plugin lainnya

💡 Best Practices yang Diimplementasikan

  • Defense in Depth: Multiple layers of security headers
  • Principle of Least Privilege: CSP restricted to minimum necessary
  • Separation of Concerns: HTML terpisah dari logic JavaScript
  • Code Maintainability: Centralized event handling untuk mudah di-audit
  • Progressive Enhancement: Fallback untuk browser old
  • No Inline Code: Semua logic di external files
